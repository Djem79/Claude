Архитектура реализации

Как лучше структурировать код для этих трёх фич

A

С рефакторингом шапки (рекомендую)

Сейчас nav-шапка продублирована в 4 файлах. Перед добавлением Dashboard вынесем её в app/admin/layout.tsx — тогда ссылку на Dashboard добавим в одном месте, а не в четырёх.

Порядок: layout.tsx → Dashboard → Kanban-тоггл → мобайл

Плюсы

  • Одно место для навигации
  • Меньше дублирования

Минусы

  • +1 задача в начале
B

Без рефакторинга — добавить фичи

Добавляем Dashboard, Kanban и мобайл, вручную обновляя nav в каждом из 4 файлов. Технический долг растёт, но работает.

Порядок: Dashboard → Kanban-тоггл → мобайл

Плюсы

  • Проще, меньше риска сломать

Минусы

  • Nav в 4 местах, сложнее менять потом
C

Kanban в первую очередь

Начинаем с самой сложной части — Kanban-тоггл в LeadsClient. Потом Dashboard. Nav обновляем в процессе.

Порядок: Kanban → Dashboard → мобайл

Плюсы

  • Главная фича готова раньше

Минусы

  • Nav всё равно придётся трогать дважды
